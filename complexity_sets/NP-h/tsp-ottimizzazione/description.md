<p>Il <strong>Travelling Salesman Problem (Ottimizzazione)</strong> chiede: date n città con distanze a coppie, trovare il tour di lunghezza minima che visita tutte le città esattamente una volta e ritorna alla partenza.</p><p>La versione ottimizzazione è NP-Hard (la versione decisionale "esiste un tour di lunghezza ≤ k?" è NP-completa). Non è noto alcun algoritmo polinomiale; i migliori algoritmi esatti sono esponenziali.</p>
